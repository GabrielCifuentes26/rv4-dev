¿Qué es el sistema?
Visión general del proyecto

El sistema Costos&Presupuestos es una aplicación web de control presupuestario para proyectos de construcción. Permite a los usuarios autorizados ver el avance de obra, monitorear presupuestos ejecutados y gestionar el equipo de trabajo.

💡
Dato clave: Es 100% web — no necesitas instalar nada. Solo entras desde un navegador (Chrome, Edge, Safari) en cualquier computadora o celular.

El sistema fue construido con tecnologías gratuitas y de código abierto:

HTML / CSS / JavaScript
Los lenguajes base de cualquier página web. HTML define la estructura, CSS el diseño visual, y JavaScript la interactividad.
Chart.js
Librería gratuita que dibuja las gráficas (barras, velocímetro) dentro del dashboard de forma automática.
Supabase
Base de datos en la nube. Guarda los usuarios, contraseñas y roles. Completamente gratuito en el plan básico.
GitHub Pages
Servicio gratuito de GitHub que publica el sitio en internet. Cualquier persona puede acceder desde su navegador.

  📁 Carpeta local              push  📦 GitHub (bodega de código)

GitHub actúa como una bodega inteligente que guarda cada versión del código. Cada vez que se hace un cambio y se sube ("push"), GitHub guarda un historial de lo que cambió y cuándo.

GitHub Pages lee esa bodega automáticamente y publica el contenido como un sitio web accesible desde cualquier navegador, sin costo alguno.

1
Se edita un archivo HTML en la computadora local
Los cambios se hacen en la carpeta del proyecto usando un editor de código.
2
Se ejecuta git push en la terminal
Este comando sube los cambios desde la computadora a la bodega de GitHub.
3
GitHub Pages detecta el cambio automáticamente
En menos de 2 minutos, el sitio en vivo se actualiza con los nuevos archivos.
4
El sitio está disponible en la URL pública
Cualquier usuario con el enlace puede ver la versión actualizada desde su navegador.

Cómo funciona el inicio de sesión
El flujo completo desde que el usuario escribe su correo

Cuando un usuario entra a login.html y escribe sus datos, ocurre la siguiente secuencia:

  [Usuario escribe correo + contraseña]
               │
               ▼
      login.html pregunta a Supabase:
      "¿Existe esta combinación?"
               │
       ┌───────┴────────┐
       ▼                ▼
   ✅  SÍ existe     ❌  NO existe
       │                │
  Crea una sesión   Muestra mensaje
  activa (token)    de error en rojo
       │
       ▼
  Redirige automáticamente
  → dashboard.html
       │
       ▼
  dashboard.html verifica:
  "¿Hay sesión activa?"
       │
  ┌────┴─────┐
  ▼          ▼
 SÍ         NO
  │          │
Muestra   Regresa a
tablero   login.html
🔑
¿Qué es una sesión? Cuando Supabase confirma que el usuario es válido, guarda una "llave temporal" en el navegador. Esta llave dura mientras el navegador esté abierto. Al presionar "Salir", la llave se destruye y hay que volver a autenticarse.

Las contraseñas nunca se guardan como texto plano. Supabase las convierte en un código cifrado (hash) que ni el administrador puede leer. Solo Supabase sabe verificar si una contraseña ingresada coincide con el hash almacenado.

Supabase tiene dos componentes que el sistema usa activamente:

Componente Función Quién lo usa

6
Roles de usuario
admin vs usuario — quién puede hacer qué

Cada usuario en el sistema tiene un rol asignado en la base de datos. El rol determina a qué partes del sistema puede acceder:

  Usuario entra con rol = "usuario"     Usuario entra con rol = "admin"
  ─────────────────────────────────     ──────────────────────────────

  ✅  Puede ver dashboard.html          ✅  Puede ver dashboard.html
  ❌  NO puede ver admin.html           ✅  Puede ver admin.html
  ❌  NO ve botón "⚙ Admin"             ✅  Ve el botón "⚙ Admin"
  ❌  NO puede crear/borrar usuarios    ✅  Puede crear, editar y
                                            eliminar usuarios

El sistema verifica el rol en dos momentos:

A
Al cargar el dashboard
El sistema consulta la tabla usuarios para saber el rol del usuario activo. Si es admin, muestra el botón "⚙ Admin" en la barra superior.
B
Al intentar entrar a admin.html
Aunque un usuario normal intente ir directamente a la URL de admin.html, el sistema verifica el rol y lo regresa al dashboard si no es admin.
Acción Rol: usuario Rol: admin
Ver index.html ✅ ✅
Iniciar sesión ✅ ✅
Ver dashboard ✅ ✅
Ver botón Admin ❌ ✅
Acceder a admin.html ❌ ✅
Crear usuarios nuevos ❌ ✅
Eliminar usuarios ❌ ✅

Panel de administrador
Qué puede hacer admin.html

El panel de administrador permite gestionar todos los usuarios del sistema sin necesidad de programar ni de pedir ayuda externa. Funciona directamente desde el navegador.

                         admin.html
                              │
           ┌──────────────────┼──────────────────┐
           ▼                  ▼                  ▼
     Ver todos         Agregar usuario     Eliminar usuario
     los usuarios           │                   │
     (tabla)                │                   │
                    ┌───────┴──────┐     ┌───────┴──────┐
                    ▼              ▼     ▼              ▼
                 En Auth       En tabla En Auth       En tabla
                 (crea la    (guarda    (borra la    (borra el
                 contraseña)  nombre    contraseña)   registro)
                              y rol)

Las operaciones disponibles en el panel son:

+
Agregar usuario nuevo
Se llena un formulario con nombre, correo, contraseña y rol. El sistema crea la cuenta en Supabase Auth y registra los datos en la tabla usuarios simultáneamente.
✎
Editar usuario existente
Se puede cambiar el nombre, correo y rol de cualquier usuario. Los cambios se guardan en la tabla usuarios de la base de datos.
✕
Eliminar usuario
El sistema pide confirmación antes de borrar. Al confirmar, elimina la cuenta de Auth y el registro en la tabla usuarios al mismo tiempo.
🔍
Buscar usuarios
Hay un campo de búsqueda que filtra la tabla en tiempo real mientras se escribe, por nombre o correo.

Diseño visual del login
Plano arquitectónico animado, animaciones y diseño responsive

La página login.html tiene un diseño visual elaborado construido completamente con CSS y SVG puro — sin imágenes externas. Cada elemento visual tiene su propósito de experiencia de usuario.

Capa de fondo — Plano arquitectónico animado:

  SVG fijo centrado en pantalla (clase .floor-plan)
  ViewBox: 0 0 900 650 px  |  Opacidad: 0.17
  ─────────────────────────────────────────────────

  15 ambientes de una casa de un nivel:
  ┌────────────────────────────────────────────────────────┐
  │  Rec. Principal  │ Baño Ppal │  Rec. 2  │   Rec. 3   │
  │    (280×240)     │ Vestidor  │ (220×160)│  (240×160) │
  │                  │ (120×120) │          │            │
  ├──────────────────┴───────────┴──────────┴────────────┤
  │              C  O  R  R  E  D  O  R                  │
  ├──────────────────────┬─────────────┬─────────────────┤
  │   SALA (380×230)     │   COCINA    │    ESTUDIO      │
  │                      │  COMEDOR    │   BAÑO VISITA   │
  ├──────────────────────┴─────────────┴─────────────────┤
  │    GARAJE 2 AUTOS (340×90)  │  TERRAZA (520×90)     │
  └─────────────────────────────┴────────────────────────┘

El plano se dibuja usando la técnica CSS stroke-dasharray / stroke-dashoffset. Cada línea, arco de puerta y mueble tiene una propiedad --len con su longitud exacta y un animation-delay escalonado:

Fase de dibujo Elementos Rango de delay
Paredes exteriores 4 líneas del perímetro 0s – 0.24s
Paredes interiores 12 divisiones de habitaciones 0.32s – 1.09s
Ventanas y puertas 10 ventanas + 11 arcos de puerta 1.16s – 2.56s
Rellenos de habitaciones 15 rectángulos dorados semitransparentes 1.0s – 2.4s
Etiquetas de texto 15 nombres de ambientes 2.3s – 3.7s
Muebles completos ~55 piezas (camas, sofás, mesas, autos, plantas…) 2.70s – 6.48s
⏱
Ciclo de animación: 18 segundos por ciclo. Todo el plano termina de dibujarse en ~10.5s y luego se mantiene visible hasta que el ciclo se reinicia. La fórmula que garantiza que ningún elemento quede incompleto es: ciclo × (1 – fracción_dibujo) > delay_máximo → 18 × 0.78 = 14.04s > 6.48s ✓

Capa de fondo — Blobs animados:

●
Blob azul (superior izquierdo)
Círculo difuminado con filter: blur(80px) que se mueve en loop con @keyframes blobMove.
●
Blob naranja (superior derecho)
Mismo efecto con delay diferente — crea movimiento asimétrico y profundidad visual.
●
Blob púrpura (centro)
Blob central más grande, da la sensación de espacio tridimensional.
●
Blob verde + blob extra (inferiores)
Completan la paleta del fondo oscuro (#050d1a).

Panel izquierdo — Encabezado animado:

El texto "Costos & Presupuestos" usa un gradiente de 5 paradas animado con background-clip: text:

  Dirección: 90deg (izquierda → derecha)
  ──────────────────────────────────────────────────
  Posición   Color           Significado
  ────────   ─────────────   ───────────────────────
    5%       #7dd3fc          Azul claro (sky-300)
   25%       #ffffff          Blanco
   50%       #D4A832          Dorado RV4
   75%       #ffffff          Blanco
   95%       #7dd3fc          Azul claro (sky-300)
  ──────────────────────────────────────────────────
  background-size: 200% auto
  animation: textShine 6s linear infinite
  "Presupuestos" usa el mismo gradiente en dirección reverse (5s)

Diseño responsive — 5 breakpoints:

Breakpoint Dispositivo destino Cambios principales
≤ 1280px Laptop / monitor pequeño Panel izquierdo padding reducido, card 480px, heading 2.8rem
≤ 1024px Tablet horizontal / laptop 13" Card 400px, heading 2.2rem, plano escala 90vw
≤ 768px Tablet vertical / móvil grande Panel izquierdo oculto, card al 100% del ancho, plano opacidad 0.13
≤ 480px Móvil estándar Padding reducido, border-radius 20px, heading 1.65rem, plano opacidad 0.10
≤ 360px Móvil pequeño (mínimo) Card mínima con padding 24px/16px, botón fuente 0.86rem
📐
Fórmula de escalado del plano: El SVG usa width: min(82vw, calc(84vh * 1.385)) donde 1.385 es el radio de aspecto 900÷650 del viewBox. Esto garantiza que el plano nunca desborde ni en ancho ni en alto, sin importar el tamaño de pantalla.

Formulario — características de UX:

①
Floating labels
Las etiquetas de los campos suben con transición CSS al escribir. También detectan autofill del navegador vía animationstart.
②
Indicador de fuerza de contraseña
4 barras de colores (rojo → naranja → verde) que evalúan longitud, mayúsculas, números y símbolos en tiempo real.
③
Ojo toggle en contraseña
Botón que alterna entre type="password" y type="text" para ver/ocultar la contraseña.
④

Destello blanco que recorre el botón de submit en loop usando @keyframes shimmer.
